Lead Discovery · Strategy & Agent Plan

Achieving full coverage of a finite TAM

Table22's ICP is roughly 13,000–18,000 US businesses across restaurants, butchers, wine retailers, bakeries, and cheese shops. Because the TAM is enumerable, this is a coverage problem, not a generation problem. The plan: 15 orthogonal discovery channels, run by a Claude agent team, into one normalized CSV schema.

The single most important reframe. The TAM is finite and enumerable. You're not generating leads from an infinite pool — you're trying to achieve completeness over ~18K businesses. The right metric is recall % per vertical per metro, not lead count. Pick three cities, build a ground-truth list manually, measure what the pipeline catches. That number governs every channel investment that follows.

07Open questions before scaling the agent team

  • Ground truth. Before running, build a hand-curated list for 3 cities × all verticals. Score each channel against it. Every channel investment is justified by recall delta.
  • Tier rubric. "Tier" in the CSV needs a shared definition across all 15 workers. Default: A = obvious premium / press-validated, B = likely premium, C = uncertain, D = filtered out.
  • Dedup key. Phone number is the existing dedup key. Workers don't always get phone numbers — fall back to (normalized name + city) and reconcile downstream.
  • Rate limiting + cost ceiling. Orchestrator must enforce per-worker call budgets to avoid runaway Serper/Apify spend.
